This code will modified in order to be merged into https://github.com/RosettaCommons/RFdiffusion. Please install the software in that repo to use the tensor outputs from make_interface_tensor.py.

These are instructions to use the script as a standalone item (rather than integrated into RFdiffusion). You will need to run these scripts in an environment with python packages os, sys, glob, torch, random, numpy, argparse, pyrosetta, and time. You'll first have to renumber the indices of your pdb if there are any gaps in the residue indices of your target protein. First, edit line 15 `pdbinfo_reset.py` to point to where you've downloaded the params file `H3i.am1bcc.fa.mm.params`. Then, renumber your pdb by running `python /path/to/pdbinfo_reset.py /path/to/alk2_target.pdb`. Here's an example command to generate some tensors used for designing the Alk2 binder: `python /path/to/make_interface_tensor.py --input_pdb /path/to/alk2_target.pdb --out_dir /path/to/ss_adjacency_tensors/ --binderlen 65 --target_adj B99-103 --binder_ss E --binder_ss_len 6`. This command should generate 59 tensors in `/path/to/ss_adjacency_tensors/`and a text list `/path/to/ss_adjacency_tensors.txt` of length 59. There are 59 tensors to sample all the locations of strand of length 6 in a binder that is 65 amino acids long.

To condition RFdiffusion with these tensors, you can run `python /path/to/rf_diffusion/run_inference.py inference.input_pdb=/path/to/alk2_target.pdb inference.output_prefix=/path/to/outputs/alk2_binder_strand_pair contigmap.contigs=[\'B66-138,0\ 65\'] ppi.hotspot_res=[\'B99\',\'B100\',\'B101\',\'B102\'] inference.model_runner=ScaffoldedSampler inference.num_designs=10 scaffoldguided.scaffoldguided=True scaffoldguided.scaffold_dir=/path/to/ss_adjacency_tensors/ scaffoldguided.scaffold_list=/path/to/ss_adjacency_tensors.txt`. Feel free to compare these interface-conditioned outputs to outputs generated without conditioning by running ` `/path/to/rf_diffusion/run_inference.py inference.input_pdb=/path/to/alk2_target.pdb inference.output_prefix=/path/to/outputs/alk2_binder_no_conditioning contigmap.contigs=[\'B66-138,0\ 65\'] ppi.hotspot_res=[\'B99\',\'B100\',\'B101\',\'B102\'] inference.num_designs=10`

The script can be used more generally for making more complex adjacencies: `python /path/to/make_interface_tensor.py --input_pdb /path/to/input.pdb --out_dir /path/to/adjacency/directory --target_adj A1-10:A50-60,A65-70:A90-90 --binder_ss H,E,E --binder_ss_len 8,8,8 --binderlen 100`. This command would generate tensors for a binder that has a Helix interface with target residues A1-10, a strand interface with A50-60 and A65-70, and another strand interface with A90. The main settings for the script are described below.  

target_adj: the blocks you want your binder secondary structures to be adjacent to, separated by ":" 
binder_ss: the list of secondary structures you want. They will be adjacent to the blocks in target adj, in the order that you give there
binder_ss_len: the length of the secondary structures in binder_ss, in respective order 
binderlen: length of your binder
